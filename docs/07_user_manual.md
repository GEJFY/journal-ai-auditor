## 1. はじめに

### 1.1 JAIAとは

JAIA（Journal entry AI Analyzer）は、仕訳データの分析と内部監査を支援するアプリケーションです。AIが自動的にリスクの高い仕訳を検出し、監査人の業務を効率化します。

### 1.2 主な機能

| 機能 | 説明 |
|------|------|
| データインポート | Excel/CSVファイルから仕訳データを取り込み |
| ダッシュボード | KPI、リスク分布、時系列分析の可視化 |
| リスク分析 | 58の監査ルールによる違反検出 |
| AI分析 | AIエージェントによる自律的な分析 |
| レポート生成 | PPT/PDF形式での監査報告書出力 |

### 1.3 画面構成

## 2. ログインと初期設定

### 2.1 アプリケーション起動

1. デスクトップの「JAIA」アイコンをダブルクリック
2. 初回起動時は初期設定ウィザードが表示されます

### 2.2 初期設定ウィザード

#### ステップ1: 会計年度の設定

```
会計年度: [2024] 年
期首月:   [4] 月
```

#### ステップ2: 重要性基準の設定

```
金額の重要性基準: [100,000,000] 円
（この金額を超える仕訳は「重要性基準超過」として検出）
```

#### ステップ3: LLM設定（オプション）

AI分析機能を使用する場合：

```
LLMプロバイダー: [Bedrock ▼]
  選択肢: Bedrock / Azure Foundry / Vertex AI / Anthropic /
          OpenAI / Google AI Studio / Azure OpenAI / Ollama

（選択したプロバイダーに応じた設定項目が表示されます）
APIキー/認証情報: [********************************]
モデル: [us.anthropic.claude-opus-4-6-20260201-v1:0 ▼]
```

## 3. データインポート

### 3.1 対応ファイル形式

| 形式 | 拡張子 | 説明 |
|------|--------|------|
| Excel | .xlsx, .xls | Microsoft Excel形式 |
| CSV | .csv | カンマ区切りテキスト |
| TSV | .tsv | タブ区切りテキスト |

### 3.2 必須カラム（AICPA GL_Detail準拠）

| カラム名 | 説明 | 例 |
|---------|------|-----|
| JE_Number | 仕訳番号 | JE-2024-00001 |
| JE_Line_Number | 明細番号 | 1 |
| Fiscal_Year | 会計年度 | 2024 |
| Accounting_Period | 会計期間 | 1-12 |
| Effective_Date | 発効日 | 2024-04-01 |
| GL_Account_Number | 勘定科目コード | 1101 |
| Amount | 金額 | 100000 |
| Entered_By | 起票者 | user001 |
| Approved_By | 承認者 | user002 |

### 3.3 インポート手順

#### ステップ1: ファイル選択

1. サイドバーの「データインポート」をクリック
2. 「ファイルを選択」ボタンをクリック
3. インポートするファイルを選択

#### ステップ2: カラムマッピング

システムが自動的にカラムを推測します：

```
┌────────────────────────────────────────────────────┐
│ カラムマッピング                                    │
├────────────────────────────────────────────────────┤
│ ファイルカラム    →    システムカラム              │
│ [仕訳番号]        →    [JE_Number ▼]               │
│ [明細番号]        →    [JE_Line_Number ▼]          │
│ [日付]            →    [Effective_Date ▼]          │
│ [勘定コード]      →    [GL_Account_Number ▼]       │
│ [金額]            →    [Amount ▼]                  │
└────────────────────────────────────────────────────┘
```

#### ステップ3: 検証とインポート

1. 「検証」ボタンをクリック
2. エラーがあれば修正
3. 「インポート実行」ボタンをクリック

## 5. リスク分析

### 5.1 違反仕訳一覧

テーブル形式で違反仕訳を表示：

| 仕訳番号 | 日付 | 勘定科目 | 金額 | リスクスコア | 違反ルール |
|---------|------|---------|------|-------------|-----------|
| JE-2024-12345 | 2024-01-15 | 売掛金 | ¥150,000,000 | 92 | 自己承認, 重要性超過 |
| JE-2024-12346 | 2024-01-16 | 現金 | ¥50,000,000 | 78 | 営業時間外 |

### 5.2 リスクスコアの計算方法

```
統合リスクスコア = ルールスコア × 0.6 + MLスコア × 0.4

ルールスコア: 各違反ルールのスコアを合算（上限100）
MLスコア: 5つの異常検知モデルの加重平均
```

### 5.4 メモ・タグ機能

#### メモの追加

1. 仕訳詳細パネルの「メモ追加」ボタンをクリック
2. メモを入力して保存

#### タグの付与

1. 仕訳詳細パネルの「タグ追加」ボタンをクリック
2. 既存タグを選択または新規作成

利用可能なタグ例：
- 要確認
- 確認済み
- 問題なし
- 追加調査必要
- 経理確認待ち

## 9. よくある質問

### Q1: データのインポートに失敗します

**A**: 以下を確認してください：
- ファイル形式が対応形式（xlsx, csv）か
- 必須カラムがすべて含まれているか
- 日付形式が正しいか（YYYY-MM-DD推奨）
- ファイルが他のアプリで開かれていないか

### Q2: AIの分析が途中で止まります

**A**: 以下を確認してください：
- LLMのAPIキーが正しく設定されているか
- インターネット接続が安定しているか
- APIの利用制限に達していないか

### Q3: リスクスコアの基準を変更したい

**A**: 設定 → 監査ルール設定から各ルールの閾値を変更できます。

### Q4: レポートに会社ロゴを入れたい

**A**: 設定 → レポート設定から会社ロゴをアップロードしてください。

### Q5: データをエクスポートしたい

**A**: ダッシュボードやリスク分析画面の「エクスポート」ボタンからCSV/Excel形式でダウンロードできます。

### Q6: 複数年度のデータを比較したい

**A**: ダッシュボードのフィルターで複数年度を選択すると、前年比較が表示されます。

### C. 用語集

| 用語 | 説明 |
|------|------|
| Benford分析 | 数値の先頭桁の分布が特定のパターンに従うという法則に基づく異常検知手法 |
| MAD | Mean Absolute Deviation（平均絶対偏差）、Benford分析の適合度指標 |
| Isolation Forest | 異常検知のための機械学習アルゴリズム |
| LOF | Local Outlier Factor、密度ベースの異常検知手法 |
| One-Class SVM | 正常データのみで学習する異常検知手法 |
| AICPA | American Institute of Certified Public Accountants（米国公認会計士協会） |
| GL_Detail | AICPA Audit Data Standardsの仕訳明細フォーマット |
