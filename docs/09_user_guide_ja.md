## 1. はじめに

### JAIAとは

JAIA（Journal entry AI Analyzer）は、仕訳データのリスク分析を自動化するAI駆動の監査支援ツールです。

### 主な機能

| 機能 | 説明 |
|------|------|
| ルールベース分析 | 58種類の監査ルールによる違反検出 |
| 機械学習分析 | 統計的異常値、クラスタリング、Isolation Forest |
| ベンフォード分析 | 数値分布の法則性チェック |
| AI対話分析 | 自然言語による質問・調査 |
| レポート生成 | PowerPoint/PDF形式の監査調書 |

## 2. 初期セットアップ

### 必要環境

- **OS**: Windows 10/11, macOS 10.15+, Linux
- **メモリ**: 8GB以上推奨
- **ディスク**: 1GB以上の空き容量

### セットアップ手順

1. **アプリケーションを起動**
   ```
   フロントエンド: http://localhost:5180
   バックエンドAPI: http://localhost:8001
   ```

2. **初回起動時のオンボーディング**
   - 自動でウェルカムウィザードが表示されます
   - 各機能の説明を確認し「次へ」をクリック
   - 「はじめる」をクリックして完了

3. **LLMプロバイダー設定（任意）**
   - 設定 → LLM設定 から選択
   - APIキーを入力
   - 接続テストで確認

## 3. データの取り込み方

### 対応フォーマット

| 形式 | 拡張子 | 備考 |
|------|--------|------|
| CSV | .csv | UTF-8推奨 |
| Excel | .xlsx, .xls | 先頭シートを読込 |

### 必須項目

| 項目名 | 型 | 説明 |
|--------|-----|------|
| journal_id | 文字列 | 仕訳ID |
| entry_date | 日付 | 仕訳日 |
| account_code | 文字列 | 勘定科目コード |
| debit_amount | 数値 | 借方金額 |
| credit_amount | 数値 | 貸方金額 |

### 取込手順

1. **「データ取込」をクリック**
   - サイドバーから選択

2. **ファイルをアップロード**
   - ドラッグ&ドロップ、または「ファイルを選択」

3. **バリデーション確認**
   - エラーがある場合は該当行が表示されます
   - 修正後、再アップロード

4. **取込実行**
   - 「取込開始」をクリック
   - 進捗バーで状況を確認

## 4. ダッシュボードの見方

### KPIカード

| カード | 説明 |
|--------|------|
| 仕訳件数 | 分析対象の総仕訳数 |
| 総取引金額 | 借方・貸方の合計金額 |
| 高リスク項目 | リスクスコア60以上の件数 |
| 自己承認 | 作成者と承認者が同一の件数 |

### エグゼクティブサマリー

画面上部のバナーに、分析結果の要約が表示されます。

### 月次推移チャート

- X軸: 月
- Y軸: 取引金額
- 期末集中や異常な変動を確認

### リスク分布チャート

- 円グラフでリスクレベル別の件数を表示
- 高リスク（赤）、中リスク（青）、低リスク（緑）

### ベンフォード分析

- 棒グラフで先頭桁の分布を表示
- 実績値（濃色）と期待値（薄色）を比較
- 適合度バッジで判定結果を表示

## 5. リスク分析の使い方

### リスクスコア

| スコア | レベル | 説明 |
|--------|--------|------|
| 80-100 | Critical | 重大な問題の可能性 |
| 60-79 | High | 要調査 |
| 40-59 | Medium | 要確認 |
| 20-39 | Low | 注意 |
| 0-19 | Minimal | 問題なし |

### フィルタリング

- **期間**: 開始日・終了日を指定
- **リスクレベル**: チェックボックスで選択
- **勘定科目**: 特定科目に絞込
- **ルールID**: 特定ルールの違反のみ表示

### 違反詳細

各違反をクリックすると：
- 仕訳の詳細情報
- 違反の説明
- 関連する他の仕訳
- 対応アクション（メモ追加、タグ付け）

## 6. AI分析機能

### 使い方

1. **「AI分析」をクリック**

2. **質問を入力**
   ```
   例: 「期末に集中している高額仕訳を調査して」
   例: 「自己承認の傾向を分析して」
   例: 「特定の勘定科目の異常を探して」
   ```

3. **分析結果を確認**
   - AIが自動で分析を実行
   - 結果と推奨アクションを表示

### 対話の継続

- 追加の質問で深掘り可能
- 「詳しく説明して」「グラフで見せて」など

### 注意事項

- LLMプロバイダーの設定が必要です
- 大量データの分析には時間がかかる場合があります

## 7. レポート生成

### 利用可能なテンプレート

| テンプレート | 説明 |
|--------------|------|
| サマリーレポート | 検証結果の概要 |
| 詳細レポート | カテゴリ別の詳細分析 |
| エグゼクティブサマリー | 経営陣向け要約 |
| 違反一覧レポート | 検出された違反の詳細 |
| リスク分析レポート | リスクスコア分布 |
| ベンフォード分析レポート | 数値分析結果 |
| 監査調書 | 監査手続と結果の記録 |

### 生成手順

1. **「レポート」をクリック**

2. **テンプレートを選択**

3. **パラメータを設定**
   - 対象年度
   - 期間範囲
   - 含める項目

4. **「生成」をクリック**

5. **ダウンロード**
   - PowerPoint (.pptx)
   - PDF (.pdf)

## 8. よくある質問

### Q: データが表示されません

**A:** 以下を確認してください：
- バックエンドが起動していますか？（画面左下の接続状態）
- データ取込は完了していますか？
- ブラウザをリロードしてみてください

### Q: リスクスコアの計算基準は？

**A:** 以下の要素で計算されます：
- ルール違反の種類と重要度
- 違反の件数
- 金額の大きさ
- 統計的な異常度

### Q: 独自ルールを追加できますか？

**A:** 設定 → ルール設定 から、しきい値のカスタマイズが可能です。完全な新規ルールの追加は、開発者による設定が必要です。

### Q: エクスポートしたレポートを編集できますか？

**A:** PowerPoint形式でエクスポートすれば、Microsoft PowerPointやGoogle Slidesで編集可能です。

### Q: 大量データを処理できますか？

**A:** 推奨は100万件以下です。それ以上の場合は、期間を分割して処理することを推奨します。
